sprite

loads ".sprite.xml" files.
(file extension is a suggestion, you may deviate from it).

a sprite sheet defines a list of animations, each of which references a texture and defines a list of frames, each of which defines a duration and a rectangle sub-area of the texture.

<sprite> single document level root tag, just to comply with picky xml text editors.
 <animation>
  name="" a unique name to identify this sprite animation by.
  texture="sprites/texture.png" a file path to a sprite sheet texture file that has all of the sprite frame images.
  font_size="10" only needed if you plan to inline the sprite in text, this tells the text layout logic how to scale the sprite. if the sprite is inlined in a font of the same size, then the sprite will display at 1:1 scale, otherwise it will scale proportionally. the individual frames in the sprite do not need to have this height.
  font_horizontal_advance="10"> only needed if you plan to inline the sprite in text, this tells the text layout logic how far to advance the text cursor after printing the sprite.
  <frame
   box="0 -10 10 0" defines the rectangular region of the sprite quad geometry around the sprite origin in pixels. if the sprite will be inlined in text, then the origin is at the cursor horizontal position (x) and the text baseline (y).
   map="0 0 10 10" defines the rectangular sub-region of the texture in pixels.
   duration="0.1"/> duration of this frame in seconds. or 0 for infinite.
  insert as many frames as you need.
 </animation>
 insert as many animations as you need.
</sprite>
