the single global engine instance is cheonsa::engine, which you can access from any .cpp file by including cheonsa_engine.h.

the engine class is the thing that ties all of the systems together, which your game interfaces with, and which manages updating and rendering of your game. there is a single global engine instance cheonsa::engine, which you can access from any .cpp file by including cheonsa_engine.h.

data, folders, files, and locale

  • [exe_folder_path] refers to the folder path of that the engine exe is executing from. this folder path is often used to find other folders.
  • [engine_data_folder_path] refers to the folder path that contains the engine data files. by default, this is expected to be at "[exe_folder_path]/data/engine/", but this can be overridden.
  • [game_data_folder_path] refers to the folder path that contains the game data files. normally for the game client this defaults to "[exe_folder_path]/data/game/". but for the game editor, loading of this folder will be deferred to when a game project is opened.
  • "engine_data_folder_path.txt" refers to the text file that is scanned for within [exe_folder_path] at start up. if it is found, then it is expected to contain a line of text, that references the path to use for [engine_data_folder_path], which if valid will override the default.

[engine_data_folder_path] and [game_data_folder_path] are both content folders, which are expected to contain a certain sub-folder structure that is tied to locale and localization. the current locale setting of the engine and the sub-folder structure of these content folders determines which files will be loaded at run time when a particuar resource is requested. this lets us publish different versions of the same file, one for each locale, as needed. this also lets us publish one version of the file to use for all locales (which is most cases).

  • "engine" contains files used by the engine.
  • "game" contains files used by the game.

additional folders may also be referenced the sub-folders in two these root folders determine what data is loaded, based on the engine's locale setting.
